# Architecture.md — 测试框架架构说明

## 项目概述

JavaTestSkeleton 是一个可复用的 Java 测试框架骨架，提供测试基础设施、代码质量检查和 CI 工作流，适用于任何 Java 17 项目。

## 模块职责

### BaseTest（测试基类）

- H2 内存数据库连接创建与销毁
- 临时目录自动管理（`@TempDir`）
- 测试配置读取（`test.properties`）
- 超时断言辅助
- SQL 执行辅助方法

### Lint 工具

| 工具 | 检查维度 | 阶段 |
|------|---------|------|
| Checkstyle | 代码风格、命名规范、格式 | `validate` |
| PMD | 未使用变量、空 catch、冗余代码 | `verify` |
| SpotBugs | 空指针、资源泄漏、并发 bug | `verify` |

### CI 工作流

5 个并行 Job：

```
unit-test ──────────┐
integration-test ───┤
checkstyle ─────────┼──► 全部通过才 ✅
pmd ────────────────┤
spotbugs ───────────┘
```

## 数据流

```
开发者 push
  │
  ▼
GitHub Actions 触发
  │
  ├──► Job 1: mvn test ──────────── 单元测试
  ├──► Job 2: mvn verify -DskipTests ─ 集成测试
  ├──► Job 3: mvn checkstyle:check ─── 代码风格
  ├──► Job 4: mvn pmd:check ────────── 静态分析
  └──► Job 5: mvn spotbugs:check ───── Bug 检测
```

## 关键设计决策

1. **Surefire vs Failsafe**：单元测试用 Surefire（`*Test.java`），集成测试用 Failsafe（`*IT.java`），互不干扰
2. **H2 内存数据库**：零外部依赖，测试数据自动销毁，CI 友好
3. **3 种 Lint 互补**：Checkstyle 管风格、PMD 管质量、SpotBugs 管 bug 模式，覆盖全面
4. **BaseTest 封装**：数据库连接、临时文件、配置读取等公共逻辑，子类只写测试
5. **并行 CI**：5 个 Job 独立运行，总耗时 = 最慢的单个 Job
